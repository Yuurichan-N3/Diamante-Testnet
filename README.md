# ⚡ Diamanteテストネット自動ファーミングスクリプト by Yuurisandesu (佐賀県産 YUURI)

## 📌 概要  
これらの2つのPythonスクリプト（`tx_V1.py`と`tx_V2.py`）は、**[Diamante Network Campaign Testnet](https://campaign.diamante.io)**で**ポイント/XPを自動的に増やす**ために作られました。  
**主な目的：** トランザクション数、XP、リーダーボードの順位を上げて、ミステリーボックスを獲得し、エアドロップ/報酬の可能性を高めること。

両スクリプトは同じ作者が作成し、バナー表示とログ記録は同じですが、**ファーミング戦略が異なります**。

---

## 🔍 両スクリプトの共通点  
✅ `.env`ファイルからプライベートキーを取得（形式：`PRIVATEKEY_1=...`、`PRIVATEKEY_2=...`など）  
✅ デバイス指紋をシミュレート（日本の位置、ユニークなdeviceId、IPの変化、緯度/経度）  
✅ `.env`からの`access-token`ヘッダーを使用（デフォルト：`"key"`）  
✅ 色付きログ + ASCIIバナー **"Yuurisandesu"**  
✅ ウォレット接続 → `userId`取得 → 繰り返し送金  
✅ レート制限（429）、無効トークン（401/403）の処理、リトライ機構  
✅ **無限ループ**で実行（**Ctrl+C**で手動停止）

---

## ⚖️ 主な違い

### 🚨 tx_V1.py（バージョン1 – **非推奨ですがファウセット効率的**）
- **戦略：** 積極的 – 各サイクルで新しいダミーアカウントを作成、リファラル`"RO5-BL18"`で登録、ファウセット請求、ダミーへ少額送金、その後固定アドレス（`0x222306e34a54455aa10c215b26aaad3d6037dbf8`）へ返送  
- **複数ウォレット：** あり、ただし最初のウォレット内で無限ループのため最適ではない  
- **送金入力：** なし（ハードコード、ウォレットごとに無限）  
- **追加機能：** ダミーアカウント登録 + 偽ツイッターハンドル + リファラルスパム  
- **モニタリング：** 基本（残高 + リーダーボードをウォレットごとに1回）  
- **クールダウン：** 固定60秒  
- **リスク：** 高い – リファラルスパムやファウセット搾取として検知されやすい  
- **推奨度：** ❌ 非推奨（古いバージョンでキャンペーン制限が厳しくなっています）

---

### ✅ tx_V2.py（バージョン2 – **推奨**）
- **戦略：** 安全＆シンプル – メインウォレットで直接ファーミング、1トークンを新しいランダムアドレスへ送金（トークン破棄）  
- **複数ウォレット：** 最適 – 各サイクルで全てのウォレットを順番に処理  
- **送金入力：** あり – ユーザーにサイクルごとの送金回数を入力させる（推奨：**10回**）  
- **追加機能：** 署名によるログイン、Cookieサポート、詳細モニタリング（ユーザーステータス、送金ごとのミステリーボックス進捗、5回送金ごとにリーダーボード更新、バッジ情報）  
- **クールダウン：** ランダム58–60秒  
- **リスク：** 低い – 通常ユーザーの行動に似ている  
- **推奨度：** ✅ **このバージョンのみ使用** – より安定して安全です

**🎯 結論：** **tx_V2.py**のみを使用してください。V1はリスクが高く、複数ウォレット操作に効果的ではありません。

## 🛠️ 使用方法チュートリアル（tx_V2.pyに焦点）

### 1. 📦 準備
- **Python 3.10**以上をインストール
- 必要なライブラリをインストール：
  ```bash
  pip install python-dotenv eth-account colorama pyfiglet faker curl-cffi
  ```

### 2. 📁 `.env`ファイルを作成（スクリプトと同じフォルダに）
  ```env
  PRIVATEKEY_1=0xabcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890
  PRIVATEKEY_2=0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef
  # 好きなだけ追加可能：PRIVATEKEY_3、PRIVATEKEY_4など
  ```

### 3. ▶️ スクリプトを実行
  ```bash
  python tx_V2.py
  ```

### 4. 📊 実行中
- スクリプトはバナーを表示し、**サイクルごとの送金回数**を入力するよう促します
- **推奨入力：** `10`（攻撃的すぎず、長時間実行に十分安全）
- **実行される処理：**
  - 全てのウォレットを接続
  - 署名でログイン
  - 残高、ステータス、リーダーボードを確認
  - 入力回数分の送金を実行
  - 条件を満たせば自動的にミステリーボックスを請求
  - 5回送金ごとにリーダーボードを更新
  - 全てのウォレット完了後 → 60秒待機 → サイクルを永久に繰り返し

### 5. ⏹️ スクリプトを停止する
**Ctrl + C**を押す（きれいに終了します）

## ⚠️ どうしてもtx_V1.pyを使いたい場合
以下の行の固定アドレスを変更：
```python
fixed = "0x222306e34a54455aa10c215b26aaad3d6037dbf8"
```
変更後：
```python
fixed = "0xYourOwnWalletAddressHere"  # 自分のウォレットアドレスに変更
```

**⚠️ 警告：** 禁止リスクが高く、複数ウォレット処理が効果的でないため、依然として非推奨です。
